This release broadens the operator surface around the agent: existing harness configs import natively, providers gain a named credential catalog with multi-active support, sandboxed spawns can run in fail-closed isolated network namespaces, and sessions expose file-change snapshots.

## Network sandboxing

- The `unshare` sandbox backend gains optional per-spawn network isolation via a new `[workspace.sandbox.network]` config block: `mode = "isolated"` runs the agent and every mediated command in a fresh deny-all network namespace.
- Network policy is delegated to an operator-supplied lifecycle provider (`<exe> setup` / `<exe> teardown`), invoked with a cleared environment and a `setns`-usable namespace handle; with no provider configured the namespace stays deny-all, not even loopback.
- Everything fails closed: provider setup failure or timeout means the workload never runs, and supervisor death tears down the provider process group and the namespace.
- `acps workspace sandbox status` reports the network mode, provider, timeout, and stderr routing; `acps workspace sandbox set` refuses to switch the backend away from `unshare` while a network block is configured.

## Agent config import

- Existing harness configs can be brought into acps: `acps agent config inspect <path>` returns a redacted review manifest (managed candidates, blocked paths, executable-settings warnings â€” never values), and `acps agent config import <path>` applies the import as a single journaled transaction.
- Selected managed provider/model/MCP fields flow into canonical acps config via repeatable `--managed-field` flags; the sanitized unmanaged residual replaces the harness's native config at its fixed destination. Unmanaged settings that can execute commands or load code require `--ack-executable-settings`.
- Supported for Claude Code, Codex CLI, OpenCode, Amp Code (MCP only), Pi (provider/model only), and Goose. Credentials, auth state, and permission/sandbox controls are always blocked from the residual.
- Admin-tier routes back the CLI (`POST /v1/agent/config/native/inspect`, `POST /v1/agent/config/native/import`, plus operation status and cancel), and hosted init accepts the same upload with a redacted review step; rollback of an applied import expires after 15 minutes.

## Providers and credentials

- The encrypted secret store now holds a provider credential catalog with named aliases, managed via `acps agent provider credential add|update|select|list|delete`. The first credential per provider is aliasless; adding a second promotes the provider to named, case-sensitive aliases. Selection is manual and target-scoped â€” no automatic failover.
- Harnesses that support it (OpenCode, Pi) can run multiple active mapped providers at once: `acps agent provider set-active <id,id,...>`, backed by `[agent.providers]` `active` and `selected_aliases` config keys.
- `acps agent provider use <id> [--model <model>]` replaces `acps agent set --provider` for mapped providers, and `acps agent provider list-active` reports configured versus last-loaded provider state. Array equivalents live under `acps array provider ... --target <id>`.
- `GET /v1/agent/status` returns sanitized `configured_providers`, `loaded_providers`, and `provider_restart_required`; legacy flat `api_key_ref` values migrate into the catalog automatically on the first provider mutation.

## Session file changes

- New route `GET /v1/sessions/{id}/changes` returns a snapshot of file changes the agent reported through ACP `diff` tool-call content, per file with `oldText`/`newText` (a `null` `oldText` marks a created file).
- The snapshot is a bounded, process-local reduction with explicit caps (7 MiB per response, 64 MiB retained overall, 512 tool calls per session) and a `truncated` flag when whole tool calls are dropped.

## Skills

- Plugin bundles are replaced by reviewed per-skill catalog indexing: repositories that organize capabilities as plugins are flattened into individually reviewed skills. `--plugins-source`/`--plugins` are removed from init, and `--skills` takes selectors that disambiguate same-name skills by repository path.
- The reviewed catalog gains new sources: `anthropic-legal`, `anthropic-finance`, `anthropic-knowledge-work`, `k-dense-scientific`, and `openai-plugins`, plus custom `github:<owner>` sources.
- Standard Setup's essential-skills step now installs Anthropic `docx`, `pptx`, `xlsx`, and `pdf` plus the OpenAI plugin skills `gh-address-comments`, `gh-fix-ci`, `github`, and `yeet`.

## Agents

- Kimi Code is wired as a native agent (`acps init --agent kimi`): managed installer, `KIMI_API_KEY` kept in the encrypted store and translated to Kimi's process-only `KIMI_MODEL_*` contract at launch, with `kimi-for-coding` as the default model.
- Known limitation: upstream Kimi Code currently rejects API-key-backed ACP sessions (`Authentication required`), so headless support is pending an upstream fix and Kimi is not yet listed in the supported-harnesses table.
