This release upgrades the ACP boundary to SDK 1.2.0. The bridge now speaks the protocol's native shapes instead of hand-rolled ones, gates prompt content and MCP transports on advertised agent capabilities, and no longer blocks the ACP event loop while a permission decision is pending.

## Protocol conformance

- The `session/fork` call uses the SDK's native `ForkSessionRequest`; the prompt breakpoint id moves from a non-conformant top-level `messageId` param to the `_meta.acpStack.messageId` vendor extension, and the fork capability probe is read from `sessionCapabilities.fork._meta.acpStack.messageId`.
- Initialization sends `clientInfo` (`name = "acp-stack"` and the running package version) and requires the agent to return protocol version 1; any other version fails the spawn before the agent becomes ready.
- Standard `usage_update` notifications (context-window `used`/`size` and cumulative `cost`) are normalized alongside the legacy token-usage shapes older agents emit, and `session_info_update` notifications patch the local session title while preserving agent timestamps and metadata.

## Capability gating

- Image, audio, and embedded-resource prompt blocks are rejected locally before dispatch unless the agent advertises the matching `promptCapabilities` flag.
- HTTP and SSE MCP declarations require `mcpCapabilities.http` and `mcpCapabilities.sse` respectively; stdio remains the unconditional baseline. Gating applies on session create, fork, load, and resume.

## Permissions

- The durable permission wait now runs off the ACP dispatch loop, so other updates and requests continue to flow while a decision is pending.
- Protocol cancellation (`$/cancel_request`) atomically cancels a still-pending permission and returns the standard request-cancelled error (`-32800`); an operator or timeout decision that wins the race is returned normally.

## MCP resolution

- Portable stdio command names are resolved to absolute executable paths before they cross the ACP boundary; session attachment fails with `config.invalid` when the command is missing or not executable.
- Native-config import validates secret references without requiring stdio executables to be installed yet, so a portable bare command can be imported before it is present on the runtime host.

## Metrics

- `GET /v1/metrics/summary` adds `usage.context_window_used_max`, the largest reported context-window usage in the requested window. Context fields use MAX because standard ACP usage updates report absolute snapshots rather than deltas; cumulative cost stays in the durable usage event rather than being aggregated across sessions or currencies.
