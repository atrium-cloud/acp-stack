Initial release of `acp-stack`, a self-hostable Linux runtime for ACP-compatible agents.

## Runtime

- ACP protocol v1 via the agent-client-protocol Rust SDK 1.1.0.
- One configured ACP agent per runtime with durable SQLite state, session create/load/resume/fork, and prompt streaming over WebSocket.
- Array mode: multiple agent targets (distinct harnesses) under a single runtime, with a primary target backing the default `agent` surfaces.
- ACP subagent wiring, agent skills install, and a pluggable agent sandbox (`off`/`unshare`/`bwrap`/`custom`).
- Supported agents: Claude Code, Codex CLI, OpenCode, Pi Agent, Goose, Amp Code, Cursor CLI (API-key auth only; headless).

## Trust layer

- age-compatible encrypted secret storage with secret references in config; plaintext never written to config.
- Tiered API keys (admin vs session), permission review for mediated commands and ACP permission callbacks, rate limits, and origin checks.
- Security self-check (`acps security check` / `GET /v1/security/check`) with persisted run history.

## Config and operations

- Portable TOML config with full export/import round-trip at `~/.config/acp-stack/acps-config.toml`.
- `acps init` guided setup (standard and advanced flows), dependency and MCP declaration validation, VM dependency profiles.
- Readiness/status endpoints, dependency checks, workspace file browse/upload/download, mediated shell commands.
- Stack self-update (`acps update`) with release manifest verification and configurable policy/frequency.

## Logging

- Local SQLite history for sessions, events, commands, and permission decisions; live WebSocket events; log filters over CLI and HTTP.
- Optional Supabase analytics mirror via a crash-safe delivery outbox.

## Packaging

- Single-binary distribution for `x86_64` and `aarch64` Linux, `install.sh` (curl-pipe and `--systemd` modes), Docker image, and systemd units including an auto-update timer.
